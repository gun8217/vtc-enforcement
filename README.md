# VTC Enforcement

승소환전소 (Victory To Cash)

Post-Judgment Enforcement Support Platform  
Web-first · Data-driven · AI-ready

---

## 1. 프로젝트 소개

**승소환전소(VTC)**는  
민사 소송에서 승소했음에도 불구하고  
집행 절차의 복잡성·비용·정보 부족으로 인해  
권리 실현을 포기하는 소액 채권자를 위한  
**판결 이후 집행 절차 지원 웹 플랫폼**입니다.

본 프로젝트는 **법률 판단을 대체하지 않으며**,  
판결 이후 단계에서 필요한 절차를  
**웹 기반으로 구조화·가시화·기록**하는 데 집중합니다.

---

## 2. 문제 정의 (Problem)

- 승소 ≠ 실제 채권 회수
- 집행 절차의 비표준화 및 정보 비대칭
- 소액 사건일수록 비용 대비 효율 부족
- 집행 결과에 대한 데이터 축적 부재

결과적으로 많은 승소자가  
판결문을 보유하고도 권리 실현을 포기합니다.

---

## 3. 해결 접근 (Approach)

### Web-first 전략

초기 단계에서는 AI 모델을 사용하지 않습니다.  
우선 다음을 목표로 합니다:

- 집행 절차의 웹 기반 표준화
- 선택 가능한 집행 옵션의 구조화
- 단계별 진행 상황 기록
- 집행 결과 데이터 축적

축적된 데이터는  
향후 자동화·지능화 기능 확장의 기반으로 활용됩니다.

---

## 4. 서비스 개요 (Service Overview)

1. 승소 판결문 업로드
2. 판결 정보 구조화
3. 집행 절차 선택지 정리
4. 단계별 집행 가이드 제공
5. 집행 관련 서류 작성 보조
6. 집행 진행 및 결과 기록
7. (선택) 외부 전문가 연계

📌 집행 여부 및 최종 판단은 이용자 또는 전문가가 수행합니다.

---

## 5. MVP 범위 (Initial MVP)

### 포함 기능

- 판결문 PDF 업로드
- 주요 항목 추출 및 정리
- 집행 절차 워크플로우 제공
- 체크리스트 기반 진행 관리
- 기본 집행 서류 자동 생성
- 집행 결과 로그 저장

### 제외 기능

- 법률 판단 자동화
- 집행 성공 보장
- 금융·자산 직접 접근
- AI 기반 추천/판단

---

## 6. 기술 구조 (Draft Architecture)

### Frontend

- 집행 절차 진행 UI
- 문서 업로드 및 상태 확인

### Backend

- 판결문 파싱 및 데이터 구조화
- 규칙 기반 집행 워크플로우
- 사용자·사건·단계 관리

### Data

- 사건별 집행 단계 기록
- 선택 옵션 및 결과 데이터
- 향후 분석을 위한 로그 축적

---

## 7. 기술 스택 (가설)

- Frontend: React / Next.js
- Backend: Node.js or Python (FastAPI)
- Database: PostgreSQL
- Document Processing: PDF Parsing
- Infra: Cloud-based (AWS/GCP 등)

※ 스택은 MVP 설계 논의 후 조정 가능합니다.
